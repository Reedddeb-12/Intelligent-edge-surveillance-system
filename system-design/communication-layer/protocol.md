# Alert Communication Protocol

This document describes how alert events generated by the edge surveillance node are structured and transmitted to the monitoring interface.

The system follows an **event-driven communication model**, where the edge device sends data only when a relevant event is detected. Instead of transmitting continuous video streams, the node sends compact alert packets containing essential information about detected objects.

This design significantly reduces bandwidth usage and allows the system to operate reliably in environments with limited network connectivity.

---

## Communication Overview

The communication pipeline works as follows:

Camera Input  
↓  
Object Detection  
↓  
Threat Classification  
↓  
Event Trigger  
↓  
Alert Packet Creation  
↓  
Transmission to Monitoring Dashboard  

When the system detects an object that satisfies the threat criteria, an alert packet is generated and transmitted to the monitoring system.

---

## Alert Packet Structure

Each alert packet contains structured information about the detected event.

Example alert packet:

```json
{
  "node_id": "edge_node_01",
  "object": "animal",
  "confidence": 0.87,
  "timestamp": "2026-03-11T19:45:00",
  "event_type": "intrusion"
}
```

---

## Packet Fields

**node_id**  
Unique identifier for the surveillance node that generated the alert.

**object**  
Type of detected object (for example: human, animal, vehicle).

**confidence**  
Confidence score produced by the object detection model.

**timestamp**  
Time at which the event was detected.

**event_type**  
Category describing the nature of the detected event.

Examples may include:

- intrusion  
- wildlife detection  
- obstacle detected  

---

## Event-Driven Communication

The system avoids continuous video transmission and instead sends alerts only when relevant events occur.

This provides several advantages:

- reduced bandwidth consumption  
- faster detection response  
- improved system scalability  
- reliable operation in low-connectivity environments  

---

## Future Communication Enhancements

Future versions of the system may incorporate **Software Defined Radio (SDR)** communication to enable long-range and resilient wireless communication between surveillance nodes and monitoring stations.

SDR-based communication can provide:

- extended communication range  
- adaptive modulation  
- improved reliability in challenging network environments
