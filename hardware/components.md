# Hardware Components

This document describes the hardware components required to build the edge surveillance node. The goal of the hardware design is to create a compact, low-power system capable of running real-time object detection in remote environments.

The surveillance node combines a camera, an edge computing device, and a solar-based power system to operate autonomously in locations where reliable power or internet infrastructure may not be available.

---

## Edge Processing Unit

**Raspberry Pi**

The Raspberry Pi serves as the primary processing unit of the surveillance node. It is responsible for running the object detection pipeline, processing camera input, and generating alert packets when potential threats are detected.

Key responsibilities:

- capturing frames from the camera module
- running the machine learning inference pipeline
- filtering detections and identifying potential threats
- generating structured alert events

The Raspberry Pi provides a good balance between computational capability, energy consumption, and ease of deployment.

---

## Camera Module

**Camera Sensor**

The camera module captures visual data from the surrounding environment and serves as the primary input source for the system.

Captured frames are continuously passed to the edge device for analysis by the object detection model.

Typical requirements for the camera include:

- sufficient resolution for reliable detection
- stable frame capture
- compatibility with the edge device

---

## Power System

To allow deployment in remote areas, the surveillance node is designed to operate using a solar-powered energy system.

The power system consists of:

**Solar Panel**

Provides renewable energy for the node during daylight hours.

**Battery Storage**

Stores energy generated by the solar panel to power the system during nighttime or low sunlight conditions.

**Power Management Circuit**

Regulates voltage and ensures stable power delivery to the edge device and camera module.

This configuration allows the surveillance node to operate continuously without relying on external power infrastructure.

---

## Optional Communication Hardware (Future Scope)

Future versions of the system may integrate additional communication hardware to improve long-range connectivity.

Possible additions include:

**Software Defined Radio (SDR)**  
Enables long-distance communication between surveillance nodes and monitoring stations.

**Wireless Communication Modules**  
Can provide connectivity using available network infrastructure where applicable.

These components would allow the system to scale into a distributed monitoring network.

---

## Hardware Design Goals

The hardware architecture is designed with the following objectives:

- low power consumption  
- reliable operation in outdoor environments  
- easy deployment in remote locations  
- compatibility with edge-based machine learning systems  

These goals ensure that the surveillance node remains practical and scalable for real-world monitoring scenarios.
