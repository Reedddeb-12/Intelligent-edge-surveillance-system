# Development Roadmap

This roadmap outlines how we plan to build and gradually deploy the Intelligent Edge Surveillance System. The goal is to move from system design to a fully functional edge-based monitoring node that can operate in remote environments with limited connectivity.

---

## Phase 1 — System Design

The first step is defining how the entire system should work. In this phase we focused on designing the overall architecture and identifying the core components required for the surveillance node.

Key steps:

- Designing the edge-based surveillance architecture
- Defining the object detection pipeline
- Identifying the hardware required for the edge node
- Designing the event-based alert mechanism
- Documenting the system workflow and data flow

Status: Completed

---

## Phase 2 — Machine Learning Pipeline

Once the architecture was defined, the next step is building the computer vision pipeline that allows the system to detect potential threats in real time.

Key steps:

- Preparing datasets for object detection
- Implementing object detection using YOLO
- Optimizing the model for edge devices
- Testing detection performance on sample inputs
- Connecting the detection pipeline with the alert generation module

Status: In Progress

---

## Phase 3 — Edge Node Implementation

In this phase we deploy the detection pipeline on an edge device and integrate it with the camera module.

Key steps:

- Setting up the Raspberry Pi environment
- Integrating the camera module
- Deploying the optimized detection model
- Processing frames in real time
- Generating alert packets when a threat is detected

Status: Planned

---

## Phase 4 — Monitoring Dashboard

To make the system useful for real-world monitoring, we plan to build a lightweight dashboard that displays alerts generated by edge nodes.

Key steps:

- Building an alert monitoring interface
- Displaying detected events in real time
- Logging timestamps and detection data
- Providing a simple interface for monitoring personnel

Status: Planned

---

## Phase 5 — Field Testing

Once the system components are integrated, the next step is testing the system in real-world conditions.

Key steps:

- Deploying the node in a controlled outdoor environment
- Evaluating detection performance
- Measuring power consumption
- Improving system stability and reliability
- Testing alert communication workflow

Status: Planned

---

## Future Enhancements

Beyond the initial prototype, we plan to explore additional improvements to make the system more scalable and robust.

Potential future work:

- Integrating Software Defined Radio (SDR) for long-range communication
- Creating multi-node surveillance networks
- Improving threat classification accuracy
- Optimizing models for lower power consumption
